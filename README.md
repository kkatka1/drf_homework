Задачи проекта

    Разработка бэкенд-сервера для SPA-приложения.

    Управление пользователями, курсами и уроками.

    Реализация CRUD-операций для работы с сущностями.

    Проверка работы эндпоинтов через Postman.

    Для сериализатора модели курса реализовано поле вывода уроков через сериализатор связанной модели.

Функциональность
Пользователи (users)

    Авторизация по email.

    Дополнительные поля:

        Телефон

        Город

        Аватарка

Курсы (courses)

    Название

    Превью (изображение)

    Описание

Уроки (lessons)

    Название

    Описание

    Превью (изображение)

    Ссылка на видео

    Привязка к курсу (один курс содержит несколько уроков)

Установка и настройка
1. Создание виртуального окружения

Для Linux/macOS:

python3 -m venv env
source env/bin/activate

Для Windows:

python -m venv env  
.\env\Scripts\activate  

2. Установка зависимостей

pip install -r requirements.txt  

3. Выполнение миграций

python manage.py migrate  

4. Запуск сервера

python manage.py runserver  

Сервер будет доступен по адресу: http://127.0.0.1:8000
Основные технологии

    Django — веб-фреймворк для разработки серверной части.

    Django REST Framework — реализация REST API.

    PostgreSQL — база данных.

    Pillow — обработка изображений.

Проверка эндпоинтов

Для тестирования используйте Postman:

    Получение, создание, изменение и удаление курсов (CRUD).

    Работа с уроками (CRUD).

Запуск проекта в Docker

    Скопируйте файл с настройками окружения:

cp .env.example .env

    Запустите все сервисы:

docker-compose up --build

    Проверка доступности сервисов:

    Веб: http://localhost:8000

    PostgreSQL: порт 5432

    Redis: порт 6379

Настройка удалённого сервера и деплой

Сервер подготовлен для развёртывания веб-приложения.

    Установлены необходимые пакеты и зависимости: Python, Django, Gunicorn, Nginx и др.

    Приложение доступно по IP-адресу сервера или домену.

    Настроены параметры безопасности:

        Закрыты ненужные порты

        Используются SSH-ключи для доступа

    Настроена автоматическая перезагрузка приложения при внесении изменений (systemd или Supervisor).

GitHub Actions workflow

Файл workflow находится в директории:

.github/workflows/

    Workflow запускается при каждом push в репозиторий.

    Включает шаг для запуска тестов проекта.

    Тесты выполняются и завершаются с отчётом.

    При ошибках тестов выполнение дальнейших шагов останавливается.

    После успешного прохождения тестов запускается деплой на удалённый сервер.

    Деплой проходит корректно, без ошибок.

Переменные окружения и секреты

    Все чувствительные данные вынесены в переменные окружения и подключены через GitHub Secrets.

    Secrets корректно используются в workflow для доступа к серверу и другим данным.
    

Git и работа с ветками

    Все изменения загружены в репозиторий.

    Задание сдано через pull request из ветки домашней работы в ветку develop.

    В коммитах отсутствуют игнорируемые файлы, такие как .env, .idea, venv, __pycache__.
